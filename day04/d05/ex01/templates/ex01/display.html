Ex01: Display process of a static page

1. URL Resolution

Django starts with the project’s main urls.py file. It checks the urlpatterns list to find a matching path. If the URL includes another app through include(), Django trims the matched prefix and continues searching in that app’s urls.py.

2. View Execution

Once Django finds the correct route, it calls the corresponding view function. For a static page, the view often uses render() to combine a request with an HTML template. 

def display(request):
    return render(request, 'ex01/display.html')

3. Template Rendering

The Django template engine loads the HTML file from the templates/ directory. It processes:

  • variable substitutions {{ variable }},
  • logic or control structures {% ... %},
  • and template inheritance or includes.

The result is a fully rendered HTML page.

4. HTTP Response

Django then wraps the rendered HTML into an HTTP response object and sends it back to the client (the browser). The browser finally interprets the HTML and displays the page.
